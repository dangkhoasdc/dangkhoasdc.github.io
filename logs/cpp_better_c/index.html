C++ functions + Template

New features in C++ (compared to C):

  1. default parameters, variable argument lists.
  2. const parameters.
  3. multiple types in a generic.
  4. operator overloading.

Example: C++ generic SUM an array

template<class T>
T sum(const T data[], int size, T s = 0) {
    for (int i = 0; i < size; ++i)
        s += data[i];

    return s;
}
  1. const: documentation, const correctness.
  2. The above function has two different signatures by matching mechanism.

Multiple Template Arguments

Example: Copy from array source to array destination.

template<class T1, class T2>
void copy(const T1 source[], T2 destination[], int size) {
    for (int i = 0; i < size; ++i) {
        // aware of `static_cast` here
        destination[i] = static_cast<T2>(source[i]);
    }
}

  • Uses polymorphism.
  • Is the heart of OOP.
  • Public Inheritance = subtyping.
    • Almost always use public inheritance.
  • Private inheritance is LIKEA relationship - not a subtype but inherits code/members turning them private.

Virtual Member Functions

  • Functions declared in the base class and redefined in the derived class.

Inheritance Mechanics

Class hierarchy defined by public inheritance creates related set of user types, all of whose objects may be pointed at by a base class pointer. By accessing virtual functions though this pointer, C++ selects the appropriate function definition at run-time.

Benefits:

  • Reuses existing code.
  • Eliminating tedious, duplicate codes.
  • Reflects relationship in the problem domain: map the real world situation into software. Understand what you are modeling [Sometimes we need somebody who know the domain]

Overloading and Function Selection

User defined types should be indistinguishable from native types

  • explicit: turn off the automatical (implicit) conversion.
  • If the operator is commutative, it is better to use friend operator than define operator inside a class since the the former is operator(a, b) and the latter means a.operator(b), respectively. The bottom line is that we should use friend operator for commutative operator, espcially when two operands are different types.

Conversion rules

  1. Use the exact match.
  2. Try standard promotions.
  3. Try standard type conversions.
  4. Try user defined conversions.

STL

Three legged stool:

These vital components were designed to:

  • work efficiently.
  • work parametrically.
  • work orthogonally.

Iterators categories

Pick the weakest iterator that lets you hava the most efficient algorithm.

Common properties:

  1. All iterators may require the * operator to dereference.
  2. To compare between two iterator, it requires the == operator and != operator.

From the weakest iterator to strongest iterator:

  1. Input: read, single pass.
    • Must allow the elements to be searched in a forward direction one-by-one.
    • Require: operator++ as well as operator== and operator!= for traserval.
    • Require: operator*.
  2. Output: write, single pass. The required operators are the same as the input iterator.
  3. Forward: read and write, multiple passes.
    • Require the ++ operator.
    • Require the * operator for dereference.
    • May be compared for equality == and !=.
    • May be obtain construction and copy construction.
  4. Bidirectional: multiple passes + two directions.
    • Require the -- and ++ operators to retreat and advance iterator, respectively.
  5. Random Access.
    • Requires effectively being able to add or substract to an iterator value and retrieve a value from the computed position. So basically, it requires all operators from a normal integral type.
    • Its values can be compared for *less and greater as well as equality operators.
