Crytocurrency Technology

Crytographic Hash Function A mathematic function: Input: any string Output: fixed-size output (usually 256 bits) Efficiently computable Properties: Collision-free It is very difficult to find $x$ and $y$ for a crytographic hash function such that: $ x\neq y {\text and} H(x)= H(y)$ What is a quantitative measures for difficult level to find a collision? One method: try $2^130$ randomly chosen inputs, we can got 99.8% chance of collision.
